CSS

  • Enter your CSS notes here
  • Adds style to a webpage. Defines how webpage content is presented to users, define style rules for specific elements of HTML page and format them w/ style options
  • Three ways to style w/ CSS:
    1. Inline CSS: useful for quick & permanent changes, not flexible
    2. Internal CSS: contains CSS rules for webpage in head section of HTML file
    3. External CSS: most preferred in most cases, allows all CSS rules in separate file, makes design changes easier
      • <link> element in <head>. ex: <link rel="stylesheet" href="./assets/style.css" />
      • rel: specifies relationship between current document & linked document
      • href: specifies location (URL) of external resource, using a relative path derived from file's current location
  • go to style.css for breakdown of selectors and declarations
  • selector: defines element or attributes to which rules, or declarations, will apply
  • declarations: two components
    1. the CSS property we want to apply
    2. the value for that property
  • margin: indicates how much space we want around the outside of an element. not visible on page, adds a buffer
  • padding: adds space around content inside an element.
  • info on box model for CSS
  • applying rule to all elements: * symbol
  • Link to MDN Web Docs on CSS to see CSS properties and values

Git

  • link to "An Introduction to Github"
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • if i need to generate more ssh keys: Github Doc Link
  • git add -A: commit all files in the working branch
  • git commit -m [insert changes to code here]: commit all changes to repo
  • git pull origin main: make sure current branch is up to date w/ main
  • git push origin [example/example]: pushes changes on github
  • If the remote branch was deleted:
    • Check your local git config if it still refers to the old-branch: git config --get-all remote.origin.fetch
    • Removing the old refs entries, can fix the fetch problem: git config --unset-all remote.origin.fetch
    • Then git fetch --prune --all

JavaScript

  • Enter your JavaScript notes here
  • Programming language that allows users to interact w/ websites
  • Can power website w/ effects, interactivity, data comm. and more
  • JavaScript basics
  • JavaScript data types and data structures
  • variable: a named container that allows data to be stored in the code
    • ex. variable statement: var shape = "circle";
  • string: this is a sequence of text known as a string. To signify that the value is a string, enclose it in single or double quote marks
  • boolean: a True/False value. The words true and false are special keywords that don't need quote marks.
  • array: a structure that allows you to store multiple values in a single reference.
    • ex: var shapes = ["triangle", "square", "pentagon", "circle"];
  • control flow: the order in which computer executes code in a script
    • in JavaScript, code runs from top to bottom.
  • conditional statements & loops: control the order in which code is executed (example in ./assets/script.js)
    • once it finds a condition that is truthy, it will run the code in the curly brackets associated with that condition, and then break out of the if statement
  • for loop: predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over
    • must use another statement (if) to interrupt control flow
  • function: a set of instructions that tells the computer how to perform a certain task
    • 2 requirements for JS function:
      1. Define the function: ex. function makeTea() {//codeblock}
      2. Call the function ex. makeTea() will execute what's in codeblock in definition
